# 🤷‍♂️ loglinter 😶‍🌫️

> Кастомный линтер для Go на базе `go/analysis`, который проверяет лог‑сообщения в коде.

# 🧾 Список выполняемых проверок:
- ✅ Сообщение должно начнаться со строчной буквы
- ✅ Сообщение не должно содержать спец. символы
- ✅ Сообщение содержит только EN символы
- ✅ Сообщение не содержит чувствительные данные
  - ❌ api_key
  - ❌ token
  - ❌ . . .
> 📚 Список чувствительных слов можно настроить через конфиг `analyzer/config_analyzer.json`
---
# ⚒️ Сборка
- Перейти в корень проекта `loglinter`
- Собрать бинарник \ В моём случае exe-шник
  - LINUX: `go build -o bin/mylinter ./cmd/mylinter`
  - WIN: `go build -o bin/mylinter ./cmd/mylinter`
> В результате появится bin/mylinter (или bin/mylinter.exe на Windows).
- Запуск линтера выполняется:
  - LINUX: `./bin/mylinter ./myproject`
  - WIN: `./bin/mylinter.exe ./myproject`
---
# ⚖️ Тестирование UNIT
> Тестовые файлы лежат в testdata/src/sample/logs.go и содержат // want "..." для проверки всех четырёх правил.
- Для запуска потребуется прописать: `go test ./...`
> Если линтер будет меняться то тесты могут упасть и показать ошибку в конкретном месте.
